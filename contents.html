X3D Specifications: Schema and DOCTYPE Validation

These assets are commonly used for XML validation of X3D scenes, and in-depth documentation is also provided here.

These X3D Specifications assets include the latest versions of recommended XML/JSON Schemas and DOCTYPEs (DTDs) for the X3D International Standards, maintained by Web3D Consortium's X3D Working Group. Technical details are found in the Recommended X3D Standards.

Autogenerated Products

The X3D schema contains complete information about the X3D scene graph and object hierarchy. Since the schema is expressed as an XML document, XSLT stylesheets can produce corresponding HTML documents or even source code for X3D APIs in full detail.

  • X3D Unified Object Model (X3DUOM) is autogenerated from X3D XML Schema.
    • Available at X3dUnifiedObjectModel-4.0.xml X3dUnifiedObjectModel-3.3.xml X3dUnifiedObjectModel-3.2.xml X3dUnifiedObjectModel-3.1.xml and X3dUnifiedObjectModel-3.0.xml
    • Initial experimentation: JSON export using Altova XMLSpy as X3dUnifiedObjectModel-4.0.json
    • Historic: originally named Object Model for X3D (OM4X3D).
    • Example simple inventory:    AllX3dElementsAttributes3.3.xml (produced by production stylesheet BuildSpecificationXmlEncodingFromSchema.xslt)
    • Example simple conversion: AllX3dElementsAttributes3.3.txt (produced by exemplar stylesheet AllX3dElementsAttributesTextTemplate.xslt)
  • X3D Java Scene Access Interface Library (X3DJSAIL) utilizes the object model as the basis for design and implementation.
    • Open-source assets include javadoc, example HelloWorldProgram.java, TODO list, classes.jar, full.jar, and source tree.
  • Support for autogenerating X3D Part 1: XML encoding specification: Clause 6 Encoding of nodes and Encoding of statements.
  • Support for autogenerating X3D Part 2: ClassicVRML encoding specification: Clause 6 Encoding of nodes.
  • Support for autogenerating X3D Java Binding specification: Annex B Node type interfaces (formerly Abstract node interfaces) and Annex C Node interfaces (formerly Concrete node interfaces).
  • Production tools: build.xml, BuildSpecificationXmlEncodingFromSchema.xslt, BuildSpecificationVrmlEncodingFromSchema.xslt and BuildSpecificationLanguageBindingJava.xslt (version control)

Design Considerations

X3D DTD. The X3D DTD (DOCTYPE) has simple expressive power.

  • Able to validate element names and attribute names are correct.
  • Able to validate element parent-child relationships are correct.
  • Able to validate DEF/USE name uniqueness (XML ID/IDREF types) are correct.
  • Attributes are treated as strings, and so attribute values usually cannot be validated as correct.

X3D Schema. The X3D Schema has significant expressive power and includes a complete object model.

  • Able to validate element names and attribute names are correct.
  • Able to validate element parent-child relationships are correct.
  • Able to validate DEF/USE name uniqueness (XML ID/IDREF types) are correct.
  • Strict typing of fields and addition of value bounds can validate attribute values are correct.
  • Node types, access types and enumeration values are defined.
  • Annotations, application information (appinfo) and documentation links provide essential guidance.
  • Object-model level of detail is sufficient to support autogeneration of comprehensive codebases and application programming interfaces (APIs).

X3D Schematron. The X3D Schematron ruleset provides additional expressive power for checking X3D-specific rules and requirements. See X3D Schematron Validation and Quality Assurance (QA) for further detail.

JavaScript Object Notation (JSON) Validation

  • coderextreme/X3DJSONLD (demonstration): X3D JSON loader and node.js express web server for serving files from localhost.
  • Manually maintained JSON schema for X3D (documentation) and possible X3DUOM autogeneration
  • X3D JavaScript Object Notation (JSON) Encoding.
  • Reference: Roy Walmsley, Don Brutzman and John Carlson, A JSON Encoding for X3D ACM SIGGRAPH Web3D 2016 Conference, Anaheim California USA, 22-24 August 2016 (presentation, .pdf).
